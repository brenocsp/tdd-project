Na imagem podemos ver o fluxo de trabalho do TDD (Test Driven Developement) ao utilizar de testes funcionais e testes de unidade. Os passos são: 

1. Escreva um teste funcional. Se o teste passar, temos que verificar se é preciso realizar a refatoração (estado verde). Se for necessário refatorar, isso deve ser feito mantendo-se o comportamento original do código, utilizando sempre teste para garantir que não quebramos nada (estado amarelo). Se não for necessário a refatoração, o processo é reiniciado, para escrever novos testes para cobrir mais partes do sistema. Se o teste não passar, realiza-se o processo da parte de baixo da imagem.

2. Escreva um teste de unidade. Se o teste passar, temos que verificar se é preciso realizar a refatoração (estado verde). Se for necessário refatorar, isso deve ser feito mantendo-se o comportamento original do código, utilizando sempre teste para garantir que não quebramos nada (estado amarelo). Se não for necessário a refatoração, voltamos para escrever novos testes para cobrir mais partes do sistema. Se o teste não passar, devemos escrever um código mínimo (estado vermelho) para que os testes passem. Devemos executar os testes novamente e voltar a verificar o ciclo já descrito.
